## Суть продукта

Проект: `Смета-Стройка AI Калькулятор`.

Это полноценный инструмент для ИЖС, домов, бань, гаражей и кровли. Пользователь создает объект на холсте или переносит его из PDF, назначает материалы и получает связанную смету по актуальным прайс-листам. В блоке КП пользователь добавляет/считает работы и формирует документ для клиента.

Первая поставка мыслится как Codex-powered коробка: папка проекта передается клиенту, запускается локально, затем совместно с клиентом настраиваются прайсы, PDF-планы, материалы, шаблоны КП и правила расчета.

Короткая логика:

> Холст дает объемы. Прайсы дают цены. Смета связывает объемы и цены. КП добавляет работы и финальную подачу.

## Важное позиционирование

Не уводить продукт в гос-сметы, ФЕР, ТЕР, ГЭСН, экспертизу и конкуренцию с ГРАНД-Сметой без прямой команды пользователя.

Продукт про коммерческий расчет частных объектов и быстрое КП.

## Приоритеты разработки

1. Полноценная структура проекта на React + TypeScript.
2. Холст/CADLight: PDF-подложка, масштаб, обводка, стены, перегородки, комнаты, проемы, этажи, гараж, баня, кровля.
3. Связанная доменная модель: каждый объект холста должен иметь расчетный смысл.
4. Прайсы: загрузка, категории, цены материалов, цены работ, поставщики, актуальность.
5. Смета: пересчет материалов на основе холста и выбранных прайсов.
6. КП: работы, условия, наценки, итоговый документ, экспорт.
7. Локальное хранение и desktop-упаковка.
8. Будущие модули: тендеры, ПТО, контроль объектов.

Первый рабочий сценарий строить вокруг ИЖС и конкретных планов/смет, загруженных пользователем. Нужны одноэтажные и двухэтажные дома.

## Технический курс

Основной стек:

- React + TypeScript + Vite.
- Tauri 2 для desktop-версии.
- SQLite для локальной базы.
- React Konva / Konva для 2D-холста.
- PDF.js для PDF-подложек.
- TanStack Table для смет, прайсов и КП.
- Zustand для локального состояния холста.
- TanStack Query для async-слоя.
- Drizzle ORM или Kysely для типобезопасной работы с БД.

Tauri закладывать как будущую desktop-упаковку. Первая рабочая версия может оставаться локальной web-версией, которую можно запускать из папки проекта.

Важно: локальная web-версия не должна полагаться только на браузерный `localStorage`. Для работы с файлами клиента нужна папка `client-data/` и локальный слой хранения: Node API на первом этапе или Tauri FS/SQLite на desktop-этапе.

Текущий CDN/Babel-прототип использовать как UX-источник, но постепенно переносить в `src/`.

## Расчетная модель

Обязательные разделы:

- фундамент/коммуникации;
- коробка;
- кровля;
- фасад/цоколь;
- террасы/крыльцо;
- теплоизоляция;
- внутренняя отделка;
- инженерные системы;
- электрика;
- сантехника;
- отопление/вентиляция;
- дополнительные работы.

Фундамент должен быть отдельным полноценным блоком: плита, лента+плита, сваи, буронабивные сваи, свайно-ростверковый сценарий, бетон, арматура, подушка, гидроизоляция, утепление, дренаж/ливневка.

Строка сметы должна хранить отдельно:

- количество;
- единицу измерения;
- цену материала;
- цену работы;
- итог материалов;
- итог работ;
- общий итог;
- источник объема;
- источник цены;
- статус проверки.

## Работа с AI

AI использовать как помощника, а не как единственный источник истины.

AI может:

- предлагать кандидатов из PDF;
- помогать максимально автоматизировать перенос PDF/чертежа на холст;
- сопоставлять прайс с материалами;
- помогать собирать актуальные прайсы по Москве и Московской области;
- находить пропущенные позиции;
- помогать формировать КП;
- объяснять расчет.

Человек должен иметь возможность подтвердить, исправить или отклонить результат.

## Правила работы с кодом

- Перед изменениями изучай существующий код.
- Давай короткий план перед крупной правкой.
- Не ломай текущий UX без необходимости.
- Делай миграцию поэтапно: сначала структура, затем доменная модель, затем перенос экранов.
- Все новые расчетные сущности типизируй.
- Если меняешь расчет, явно указывай формулы и допущения.
- После задачи пиши, какие файлы изменены и что проверить руками.
- Перед добавлением сложной автоматизации проверяй, не упирается ли она в ограничения браузера, файловой системы, Codex или локального запуска.

## Термины

- КП - коммерческое предложение.
- Холст - 2D-рабочая область, где создается объект.
- Обводка - ручной перенос PDF/картинки на расчетный холст.
- Кандидаты - элементы, которые система предложила из PDF и ждет подтверждения.
- Технологический пакет - набор материалов, работ, формул и правил для конструктивного элемента.
- Коробочное внедрение - персонализированная версия продукта под конкретную компанию.
